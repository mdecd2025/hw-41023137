1.請用繁體中文詳細說明下列資料: https://cyberbotics.com/doc/guide/introduction-to-webots 

什麼是 Webots？

Webots是一款專業的行動機器人模擬軟體套件。它提供了一個快速原型環境，讓使用者可以創建具有質量、關節、摩擦係數等物理特性的 3D 虛擬世界。

使用者可以添加簡單的被動物件或稱為移動機器人的主動物件。這些機器人可以有不同的運動方案（輪式機器人、腿部機器人或飛行機器人）。此外，它們可能配備有許多感測器和執行器裝置，例如距離感測器、驅動輪、攝影機、馬達、觸控感測器、發射器、接收器等。

最後，使用者可以單獨對每個機器人進行編程，以表現出所需的行為。 Webots包含大量的機器人模型和控制器程式範例，幫助使用者入門。

Webots 還包含許多與真實移動機器人的接口，因此一旦您的模擬機器人表現如預期，您就可以將其控製程序轉移到真實的機器人，如 e-puck、DARwIn-OP、Nao 等。

Webots 模擬：

Webots 模擬由以下項目組成：

定義一個或多個機器人及其環境的 Webots 世界檔案 (.wbt)。 .wbt 檔案有時確實依賴外部 PROTO 檔案 (.proto) 和紋理。

一個或多個用於上述機器人的控制器程式（C/C++/Java/Python/MATLAB 語言）。

可選的實體插件，可用於修改 Webots 常規物理行為（在 C/C++ 中）。

什麼是世界？

在 Webots 中，世界是對機器人及其環境屬性的 3D 描述。它包含每個物件的描述：位置、方向、幾何形狀、外觀（如顏色或亮度）、物理屬性、物件類型等。

世界被組織為層次結構，其中物件可以包含其他物件（類似於 VRML97）。

例如，一個機器人可以包含兩個輪子、一個距離感測器和一個本身包含攝影機的關節等。

世界文件不包含機器人的控制器程式碼；它僅指定每個機器人所需的控制器的名稱。世界保存在“.wbt”文件中。 「.wbt」檔案儲存在每個 Webots 專案的「worlds」子目錄中。

什麼是控制器？

控制器是一種控制世界文件中指定的機器人的電腦程式。

控制器可以用 Webots 支援的任何程式語言編寫：C、C++、Java、Python 或 MATLAB。

當模擬開始時，Webots 會啟動指定的控制器（每個控制器作為單獨的進程），並將控制器進程與模擬機器人關聯起來。

請注意，多個機器人可以使用相同的控制器程式碼，但是每個機器人都會啟動不同的進程。

有些程式語言需要編譯（C 和 C++），其他語言需要解釋（Python 和 MATLAB），有些需要編譯和解釋（Java）。

例如，C 和 C++ 控制器被編譯為平台相關的二進位可執行檔（例如 Windows 下的“.exe”）。

Python 和 MATLAB 控制器由對應的執行時間系統（必須安裝）解釋。

Java 控制器需要編譯為字節碼（“.class”檔案或“.jar”），然後由 Java 虛擬機器解釋。

每個控制器的來源檔案和二進位檔案都一起儲存在一個控制器目錄中。每個 Webots 專案的「控制器」子目錄中都會有一個控制器目錄。

什麼是主管控制器？

主管控制器是主管欄位設定為 TRUE 的機器人的控制器，它可以執行通常只能由人類操作員而不能由真實機器人執行的操作。

Supervisor 控制器可以用上述任何一種程式語言編寫。然而，與常規機器人控制器不同的是，Supervisor 控制器可以存取特權操作。特權操作包括模擬控制，例如將機器人移動到隨機位置、對模擬進行視訊擷取等。

文法說明：

第一段：What is Webots?
Webots is a professional mobile robot simulation software package.
主詞： Webots

be 動詞： is

補語（名詞片語）： a professional mobile robot simulation software package

用法說明： 簡單現在式，用來描述「Webots 是什麼」。

It offers a rapid prototyping environment, that allows the user to create 3D virtual worlds...
主詞： It（= Webots）

動詞： offers

受詞： a rapid prototyping environment

關係子句： that allows the user to create...

語法重點： 這是一個限定用法的關係子句，用 that 引導（非用逗號比較正確，這裡有些文體鬆散）。

These robots can have different locomotion schemes (wheeled robots, legged robots, or flying robots).
情態動詞： can（表示能力或可能性）

have + 名詞片語： have different locomotion schemes

括號補充說明： 非限定用法，用來列舉例子（可省略不影響主句結構）

They may be equipped with a number of sensor and actuator devices...
may + 被動語態： may be equipped

with + 名詞片語： with sensor and actuator devices

語法重點： 被動語態，表示「可能會被配備」

第二段：What can I do with Webots?
Webots is well suited for research and educational projects...
be 動詞 + 過去分詞： is suited（表被動狀態）

well 修飾 suited： 程度副詞

for + 名詞片語： for research and educational projects

Many mobile robotics projects have relied on Webots for years...
現在完成式： have relied（從過去到現在一直使用）

on + 受詞： on Webots

for + 時間： for years（表示持續一段時間）

段落：What is a Controller?
A controller is a computer program that controls a robot specified in a world file.
主句： A controller is a computer program

關係子句： that controls a robot

分詞片語修飾 robot： specified in a world file（過去分詞，修飾 robot）

語法重點： 關係子句 + 分詞修飾，表明是哪一個機器人

Some programming languages need to be compiled (C and C++), other languages need to be interpreted...
need to + 被動動詞： need to be compiled / interpreted

列舉對比結構： some..., other...

括號中的例子為補充說明

Supervisor Controller 段落
The Supervisor controller is the controller of a Robot whose supervisor field is set to TRUE...
主句： The Supervisor controller is the controller

of a Robot（介系詞片語）

whose... is set to TRUE（關係子句）： 修飾 Robot

語法重點： whose 引導所有格關係子句

3.請摘要出下列內容:

What is Webots?
Webots is a professional mobile robot simulation software that provides a rapid prototyping environment. It allows users to create 3D virtual worlds with physics properties (e.g., mass, joints, friction) and add passive or active objects like mobile robots. Users can equip robots with sensors and actuators and program them for desired behaviors. It offers numerous robot models, examples, and interfaces to transfer simulations to real robots like e-puck or Nao.

What can I do with Webots?
Webots is ideal for research and education in mobile robotics, supporting projects like robot prototyping (e.g., academic, automotive, toy industries), locomotion research (legged, humanoid robots), multi-agent studies (swarm intelligence, collaborative robots), and adaptive behavior research (genetic algorithms, AI). It’s also used for teaching robotics and robot competitions.

What do I need to know to use Webots?
Basic knowledge of C, C++, Java, Python, or MATLAB is required to program controllers, though beginners can use the graphical BotStudio language for certain robots (e.g., e-puck). To create custom models or environments, familiarity with 3D graphics and VRML97 is needed for designing or importing 3D objects.

How do I get User Support?
Cyberbotics offers paid support via Premier Service plans and custom development. Community-based support is available through Robotics StackExchange (webots tag), GitHub Discussions (development questions), GitHub Issues (bug reports), and Discord (quick discussions), though responses aren’t guaranteed.

Webots Simulation
A Webots simulation consists of a world file (.wbt) defining robots and their environment, controller programs (in C/C++/Java/Python/MATLAB), and an optional physics plugin (in C/C++) to adjust physics behavior. World files may rely on external PROTO files and textures.

What is a World?
A "world" in Webots is a 3D description of robots and their environment, detailing properties like position, geometry, appearance, and physics. Organized hierarchically (like VRML97), world files (.wbt) specify controller names but don’t include the controller code itself.

What is a Controller?
A controller is a program that directs a robot in a world file, written in C, C++, Java, Python, or MATLAB. Webots launches each controller as a separate process per robot at simulation start. Some languages require compilation (e.g., C/C++), others interpretation (e.g., Python), with files stored in the "controllers" subdirectory.

What is a Supervisor Controller?
A supervisor controller, tied to a robot with its "supervisor" field set to TRUE, can perform privileged actions (e.g., simulation control, moving robots, recording videos) typically reserved for human operators. Written in any supported language, it has greater capabilities than standard controllers.

  • 原文句： "...that allows the user to create 3D virtual worlds with physics properties..."

  • 中文： 物理屬性

  • 原文句： "...with physics properties such as mass, joints, friction coefficients..."

  • 中文： 真實的

  • 原文句： "...you can transfer its control program to a real robot like e-puck..."

  • 用法： 修飾 robot，表示與模擬相對的真實機器人。


10. adaptive (adj.)

  • 中文： 適應性的

  • 原文句： "Adaptive behavior research (genetic algorithm, neural networks...)"

5.請詳細說明如何採手動方式在 Webots 建立下列場景:


步驟一：啟動 Webots 並建立新的專案
打開 Webots。

點選上方選單：
File → New Project Directory...
輸入資料夾名稱，按下「Create」。

接著會跳出建立場景的選項，選擇 Empty World → 點選「Finish」。

步驟二：建立世界元素（地板、物體、機器人等）
➤ 建立地板（Ground / Floor）
在左側 "Scene Tree"（場景樹） 視窗中，選取 root。

右鍵 → Add new node...
選擇 Solid → OK。

在 Solid 節點下，再加入：

Shape（幾何外型）

Appearance → Material（材質）

Box（地板的形狀）

設定屬性如下（右側選單）：

size: 10 10 0.1（寬、長、高）

translation: 0 0 0.05

小技巧：你也可以使用 Floor 節點，它是預設的地面。

➤ 加入障礙物（例如牆壁、箱子）
點 root → 右鍵 → Add new node... → 選擇 Solid。

在其中加入：

Shape → Box

Appearance（可加材質）

設定：

size: 0.5 0.5 0.5（障礙物大小）

translation: 1 1 0.25（位在地面上的座標）

➤ 加入機器人（e-puck）
點 root → 右鍵 → Add new node...

搜尋並加入：E-puck（Webots 內建機器人）

設定位置：

translation: -1 0 0.035（剛好放在地上）

步驟三：加入感測器或照明（選擇性）
➤ 加入光源（Light）
點選 root → Add new node... → 選擇 PointLight 或 DirectionalLight

設定位置與亮度，例如：

translation: 0 0 5（上方照射）

color: 1 1 1（白光）

➤ 加入相機（Camera）
如果你想讓機器人視角觀看世界：

點選機器人（例如 E-puck）節點。

右鍵 → Add new node... → 選擇 Camera

設定解析度與啟用時間：

width: 640

height: 480

在控制器中啟用 camera（見程式控制方式）

步驟四：儲存與執行
點選上方選單 File → Save World As...
存成 .wbt 檔案，放在 worlds 資料夾中。

點擊播放按鈕 ▶️ 開始模擬！

延伸：讓機器人移動（選擇性）
如果你想測試機器人移動，可加入控制程式：

點選機器人節點 → 設定 controller 欄位（例如選 e-puck_line_following）。

或者自己寫一個簡單 Python 程式，放在 controllers 資料夾中。

6.希望可以手把手一步步說明該如何利用圖形介面 menu 建立這個場景, 過程中有那些要點必須特別注意?

一、建立新的模擬專案與場景
步驟 1：開啟 Webots → 新建專案
開啟 Webots。

點選上方選單：
File → New Project Directory...

輸入新專案資料夾名稱，例如 my_scene → 點「Create」。

選擇：
Empty world（空白場景）→ 點「Finish」。

你現在會進入一個空白場景，只有地面與天空。

二、加入地板（Floor）
步驟 2：加入地板（Box 形式）
在左邊的 Scene Tree（場景樹）中，右鍵點擊 root → 選擇：
Add New Node...

搜尋並選擇：Solid → 點「Add」

選中剛剛新增的 Solid，右鍵 →
Add New > Shape

在 Shape 底下右鍵 →
Add New > Box

再對 Shape 右鍵 →
Add New > Appearance → Material

重點設定（右側「Field」欄位設定）：

在 Box 的 size 欄位填入：10 10 0.1

在 Solid 的 translation 欄位填入：0 0 0.05

說明：

地板厚度是 0.1，所以 y 軸（高度）要設定 0.05 讓它貼齊地面。

這樣你就建立了一個10x10的「平面地板」。

三、加入障礙物（例如箱子或牆壁）
步驟 3：新增障礙物
對 root 右鍵 →
Add New Node... → 選 Solid

在此 Solid 裡一樣新增：

Shape → Box

Appearance → Material

設定數值如下：

Box > size: 0.5 0.5 0.5

Solid > translation: 1 1 0.25

說明：

高度為 0.5 → 要設在 y 軸 0.25 才能「站在地上」

四、加入機器人（E-puck）
步驟 4：加入 e-puck 機器人
對 root 右鍵 →
Add New Node...

搜尋並選擇：E-puck → 點「Add」

設定位置：

在右側欄位中，找到 translation → 設定為：-1 0 0.035

說明：

e-puck 機器人本身高度是 0.07 左右，因此高度設為一半 0.035 會剛好落在地板上。

五、加入光源（DirectionalLight）
步驟 5：讓場景更亮
對 root 右鍵 →
Add New Node...

搜尋：DirectionalLight → 點「Add」

設定建議：

direction: -1 -1 -1（斜向照下來）

color: 1 1 1（白色光）

intensity: 2（亮一點）

六、加入攝影機（選擇性）
步驟 6：把相機加在 e-puck 上
點開 E-puck → 找到其 children 節點 → 右鍵 →
Add New Node...

選擇：Camera → 點「Add」

建議設定：

width: 640

height: 480

translation: 0 0 0.05（放在機器人頭上）

七、儲存並執行模擬
步驟 7：儲存場景
點選上方選單：
File → Save World As...

命名檔案，例如 basic_scene.wbt → 儲存到 worlds/ 資料夾

步驟 8：執行模擬
點選工具列上方的「播放」按鈕 ▶️

機器人會初始化（雖然還沒動，除非你加控制器）
